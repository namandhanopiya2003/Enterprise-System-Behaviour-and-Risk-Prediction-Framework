# ABOUT THIS FOLDER: Case Study A - System Behavior Modeling

This folder contains all files created for Case Study A. The objective of this case study was to analyze how an existing enterprise client evolved from a single-brand setup to a multi-brand environment, understand how business and technical decisions influenced system behavior, and identify recurring patterns that emerged over time.<br>
The analysis is organized into four layers:

## CaseStudyA - LAYER 1 - ANALYSIS

This file contains the structured datasets used to capture events, business context, impacts, decisions, system evolution, and supporting analysis. It serves as the foundation for all subsequent layers.<br>
Live File Link: https://docs.google.com/spreadsheets/d/1iQS7ziB1GUQysUczPsK1O7blnR_0VtEDsKMry9OFjUI/edit?usp=sharing

---

## CaseStudyA - LAYER 2 - REPORT

This file converts the structured analysis into a chronological narrative, explaining what happened, why decisions were made, the trade-offs involved, and how the system evolved throughout the implementation journey.<br>
Live File Link: https://docs.google.com/document/d/1nGmsKOR4k_T7Zl1LIjtaHT6GBsm3-aSmpePM3xF50kQ/edit?usp=sharing

---

## CaseStudyA - LAYER 3 - VISUALIZATION

This file contains dashboards and visual analysis created from the structured datasets. The visualizations highlight trends, dependencies, risk areas, system evolution, and other key insights identified during the analysis.

---

## CaseStudyA - LAYER 4 - SYSTEM INTELLIGENCE

This file contains the intelligence extracted from the case study, including event relationships, recurring patterns, behavior models, root cause analysis, and system-level insights. The outputs from this layer were later used to generate predictions in Case Study B.<br>
Live File Link: https://docs.google.com/spreadsheets/d/1C-vX_-jPfpCnPo6sWPZQ57LPmDxVLkfP3dSOmSxDSlo/edit?usp=sharing
